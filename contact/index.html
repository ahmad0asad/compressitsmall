What we can help with

Tool usage questions, workflow recommendations (e.g., “make this PDF under 2MB”), bug reports, and feature requests that improve the quality of the toolkit.

If you are unsure which tool to start with, include your target platform (email, job portal, scholarship portal), the file type, and the size limit you must meet.

What to include in your message

Required: tool URL, browser + version, operating system, and a short step-by-step description.

Helpful: file type, approximate size, page count, and whether the file is scanned or text-based.

Security note: Please do not email confidential documents. If needed, reproduce the issue with a non-sensitive sample file.

Response expectations

Support is best-effort. For urgent or business-critical workflows, always keep an original copy of your file and verify results before submission.

If a tool cannot reliably achieve a requested conversion in-browser (for example, complex Office-to-PDF conversions), we will recommend safer, standard alternatives.

Need Help? What to Include When You Contact Us

Please do not email sensitive documents. Instead, share the tool URL, your browser/device, and what error you see.

  • Tool page URL
  • Device + browser
  • File type + approximate size
  • Steps to reproduce the issue

Self‑help: Upload Limit Checker and File size checklist.

How to get the best support

Compress It Small is a small, independently run project. Response times are typically within 1–3 business days. To help resolve your query as quickly as possible, please include specific details when contacting us.

For tool issues

Include the following in your message: the exact tool URL (for example, compressitsmall.co.uk/tools/pdf/compress-pdf/), the browser name and version (Chrome, Firefox, Safari, Edge), your operating system (Windows, macOS, iOS, Android), the approximate file size and type, and a short description of what happened versus what you expected. This allows us to reproduce the issue accurately and respond with a specific fix or workaround.

For feature requests

We genuinely read and consider feature requests. Common requests already planned: batch processing (compressing multiple files at once), OCR for scanned PDFs, and additional image format support. If you have a different request, describe the exact workflow problem you are trying to solve and we will assess whether it fits the scope of the project.

For privacy and data questions

All tools on this site process files locally in your browser — your files are never sent to our servers. If you have a question about a specific tool's privacy behaviour, reference the tool by name and URL and we will confirm exactly what data, if any, is communicated externally.

Contact email

You can reach us at: compressitsmall@gmail.com

Please use a clear subject line such as "Tool issue — Compress PDF" or "Feature request — batch processing" to help us route your message quickly.

Common issues with quick self-service fixes

  • Compressed PDF still too large — See making a PDF under 2 MB and try Split PDF.
  • Tool produced blank or corrupt output — Try in a private/incognito window with extensions disabled. Browser extensions frequently interfere with file processing tools.
  • File rejected by a portal despite being the right size — Check our Upload Limit Checker and the upload limits guide.
  • Broken link or page error — Include the URL of the page you were on and the URL the broken link pointed to.
